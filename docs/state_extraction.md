# Извлечение состояния стран и приоритизация обновлений

## 1) Перечень извлекаемых сигналов

При каждом обновлении состояния страны система извлекает и нормализует следующие сигналы.

### 1.1 Войны
- Факт участия в войне (да/нет).
- Роль в конфликте: инициатор, обороняющийся, союзник.
- Количество активных фронтов.
- Динамика: эскалация/деэскалация за последние N дней.
- Потери (личный состав/техника) и темп потерь.

### 1.2 Сила армий
- Суммарная военная мощь (композитный индекс).
- Численность сухопутных сил, авиации, флота.
- Мобилизационный потенциал и текущий уровень готовности.
- Качество армии (доктрина, опыт, снабжение).
- Тренд изменения силы (рост/падение относительно последнего окна).

### 1.3 Торговая зависимость
- Доля импорта критических ресурсов.
- Концентрация поставщиков (риск зависимости от 1–2 стран).
- Экспортная уязвимость (доля доходов от ключевых рынков).
- Зависимость по логистическим маршрутам/узким местам.
- Чувствительность к санкциям и блокадам.

### 1.4 Территориальные претензии
- Активные претензии на регионы/провинции.
- Взаимные претензии и степень конфликта по ним.
- Исторические/идеологические обоснования (если доступны).
- Статус урегулирования: заморожено, переговоры, эскалация.

### 1.5 Текущие кризисы
- Тип кризиса: военный, политический, экономический, гуманитарный.
- Интенсивность кризиса (низкая/средняя/высокая/критическая).
- География и вовлечённые страны.
- Временной профиль: новый, затяжной, на спаде.

### 1.6 Отношения по 14 параметрам
Для пары стран хранится вектор из 14 параметров (например, шкала -100..+100):
1. Дипломатическое доверие.
2. Идеологическая совместимость.
3. Торговая интеграция.
4. Военное сотрудничество.
5. Разведывательное взаимодействие.
6. История конфликтов.
7. Пограничная напряжённость.
8. Санкционное давление.
9. Энергетическая зависимость.
10. Конкуренция за влияние в регионе.
11. Поддержка в международных институтах.
12. Миграционно-гуманитарная связность.
13. Информационная/пропагандная враждебность.
14. Риск нарушения договорённостей.

## 2) Формат «краткой памяти» страны

Краткая память хранит недавние события, влияющие на интерпретацию текущего состояния.

```yaml
country_memory:
  country_tag: "GER"
  updated_at: "1939-08-25T12:00:00Z"
  recent_deals:
    - id: "deal_001"
      counterpart: "SOV"
      type: "trade|non_aggression|military_access|guarantee"
      date: "1939-08-23"
      significance: 0.82
      expires_at: "1940-08-23"
  betrayals:
    - id: "betr_004"
      counterpart: "POL"
      date: "1939-09-01"
      domain: "military|diplomatic|economic"
      severity: 0.93
      unresolved: true
  aid_events:
    - id: "aid_017"
      actor: "USA"
      recipient: "ENG"
      kind: "lend_lease|volunteers|intel|financial"
      date: "1940-05-12"
      amount_index: 0.61
  sanctions:
    - id: "san_005"
      sender: "ENG"
      date: "1939-09-03"
      scope: "oil|steel|finance|full"
      pressure: 0.74
  secret_pacts_risk:
    score: 0.68
    confidence: 0.55
    drivers: ["troop_movements", "trade_anomalies", "intel_reports"]
  territorial_demands:
    - target: "DANZIG"
      basis: "historical|strategic|ethnic"
      urgency: 0.88
      escalation_risk: 0.79
```

### 2.1 Семантика полей
- `recent_deals`: последние значимые соглашения и их «срок действия» в памяти.
- `betrayals`: факты нарушения обязательств; повышают риск эскалации.
- `aid_events`: события помощи, меняющие баланс сил.
- `sanctions`: санкционные меры и их интенсивность.
- `secret_pacts_risk`: агрегированный риск скрытых договорённостей.
- `territorial_demands`: активные требования по территориям и их срочность.

Рекомендуемое окно хранения краткой памяти: 180–360 игровых дней (с экспоненциальным затуханием веса).

## 3) Приоритизация стран (Tiering)

### Tier 1 — максимальный приоритет
Включает:
- Игрока.
- Великие державы.
- Любые страны, вовлечённые в активные кризисы.

Политика обновления:
- Частота: каждый тик/каждый день (в зависимости от модели симуляции).
- Полный пересчёт всех сигналов и relation-вектора.
- Низкий порог значимости для генерации событий.

### Tier 2 — средний приоритет
Включает:
- Региональные силы.
- Страны, влияющие на баланс в регионе, но вне глобального кризиса.

Политика обновления:
- Частота: реже (например, раз в 3–7 дней).
- Частичный пересчёт «тяжёлых» метрик по расписанию.
- Средний порог значимости.

### Tier 3 — фоновый приоритет
Включает:
- Второстепенные страны.
- Страны без существенной внешнеполитической динамики.

Политика обновления:
- Частота: редко (например, раз в 14–30 дней) или по триггеру.
- Обновление по событиям-исключениям (война, переворот, санкции, претензии).
- Высокий порог значимости для создания новых событий.

### 3.1 Динамическое повышение приоритета
Страна может временно подниматься на Tier выше, если:
- начинается война;
- возникает новый кризис высокой интенсивности;
- фиксируется резкий скачок по relation-параметрам или силовому балансу.

## 4) Cooldown-политика и пороги значимости событий

### 4.1 Базовая cooldown-политика
Чтобы избежать «спама» событий, используется cooldown по типам.

Пример базовых значений:
- Военные события: 3–7 дней.
- Дипломатические события: 5–10 дней.
- Экономика/торговля/санкции: 7–14 дней.
- Территориальные претензии: 10–20 дней.
- Повтор одинакового сюжетного шаблона: +50% к текущему cooldown.

Дополнительно:
- Для Tier 1 cooldown уменьшается на 25%.
- Для Tier 3 cooldown увеличивается на 50%.
- При критическом кризисе cooldown может быть временно отключён для emergency-событий.

### 4.2 Пороги значимости
Событие генерируется, если агрегированная значимость `S` превышает порог Tier.

```text
S = w1*Δwar + w2*Δarmy + w3*Δtrade + w4*Δclaims + w5*Δcrisis + w6*Δrelations + w7*memory_impact
```

Где:
- `Δ*` — нормализованные изменения сигналов за окно наблюдения.
- `memory_impact` — вклад краткой памяти (предательства, санкции, помощь и т.д.).
- `wi` — настраиваемые веса доменов.

Рекомендуемые стартовые пороги:
- Tier 1: `S >= 0.35`
- Tier 2: `S >= 0.50`
- Tier 3: `S >= 0.70`

Усиливающие коэффициенты:
- +0.10 к `S` при активной войне.
- +0.08 при наличии свежего `betrayal` (<30 дней).
- +0.05 при новых территориальных требованиях высокой срочности.

### 4.3 Антидребезг (hysteresis)
Для стабильности рекомендуется:
- Подтверждать генерацию события, если порог превышен 2 апдейтами подряд.
- Использовать «зону тишины» после события (grace period), где требуется +0.05 к порогу.

### 4.4 Логирование для дебага
Для каждого решения логировать:
- текущий Tier;
- значение `S` и вклад компонентов;
- активный cooldown;
- причину генерации/отклонения события.

Это упрощает настройку баланса и анализ ложных срабатываний.
