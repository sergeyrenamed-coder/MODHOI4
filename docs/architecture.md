# MODHOI4 Architecture

## 1) HOI4 Mod Layer
- Слой игровых скриптов и контента (фокусы, решения, ивенты, on_actions, scripted_*).
- Назначение слоя: только безопасно применить уже рассчитанное внешним AI решение в игровом контексте.
- В этом слое не формируется логика поведения AI; он исполняет готовый результат.

## 2) State Extraction
- Слой извлечения состояния игры в структурированный формат.
- Снимает необходимые переменные/флаги/контекст и готовит пакет данных для внешней обработки.
- Использует нейминг-конвеншн:
  - flags: `aidip_flag_*`
  - variables: `aidip_var_*`
  - events: `aidip.<id>`

## 3) AI Bridge
- Слой интеграции с внешним AI-сервисом (transport, validation, mapping команд).
- Получает state-пакет из State Extraction и возвращает набор применимых действий.
- Возвращаемые действия исполняются через HOI4 Mod Layer как shell-механика.

## Naming Convention (Single Standard)
- Все новые сущности мода должны использовать префикс `aidip_`.
- Флаги: `aidip_flag_*`
- Переменные: `aidip_var_*`
- Ивенты: `aidip.<id>`
